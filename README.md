# barktown

A **static audio-diary viewer** for a collection of `.m4a` / `.aac` voice recordings.
No server, no database. Everything runs in the browser from pre-built
static files you generate on your own machine. Audio and waveform assets are
hosted on an S3-compatible object store (MinIO).

---

## Purpose

barktown turns a folder of date-stamped `.m4a` recordings into a scrollable
24-hour timeline — one row per day, pins for each recording. Click a pin to open
a waveform player panel. The pipeline is two steps:

1. **Preprocess** — run a Node.js script once (or after adding new recordings).
   It reads duration via `ffprobe`, generates waveform peak data via
   `audiowaveform`, and writes a single `index.json` manifest.
2. **Serve** — build the SvelteKit site (`npm run build`) and open `build/` in
   any static file host (Caddy, nginx, `npx serve`, GitHub Pages, …).

`s3.angiehjort.com/barktown/`, organised by date:

```

│       └── MM/

└── waveforms/
```

## Quick start

### 1 — Prerequisites

```bash
# macOS (Homebrew)
brew install ffmpeg audiowaveform

# Debian / Ubuntu
sudo apt install ffmpeg
# audiowaveform: see https://github.com/bbc/audiowaveform#installation
```

### 2 — Install JS dependencies

```bash
npm install
```

### 3 — Add recordings

Files must be placed in the S3 bucket under `audio/YYYY/MM/` and follow the pattern:

```
YYYY-MM-DD HH-MM-SS optional free text.m4a
YYYY-MM-DD HH-MM-SS optional free text.aac
```

Examples:
```
audio/2026/02/2026-02-07 17-25-00 barks.m4a
audio/2026/01/2026-01-11 12-41-00 barks and yaps.m4a
audio/2025/12/2025-12-11 05-32-00.aac
```

Files that don't match the pattern are skipped with a warning.

### 4 — Preprocess

```bash
npm run preprocess
# or explicitly:
node scripts/build-audio-index.mjs
```

This writes `index.json` and `waveforms/YYYY/MM/*.json` (upload both to S3).

### 5 — Develop

```bash
npm run dev
```

Opens a hot-reloading dev server at `http://localhost:5173`.

### 6 — Build

```bash
npm run build
```

Produces a fully self-contained static site in `build/`. Serve it with:

```bash
npm run preview          # local preview of the production build
npx serve build          # or any static file server
```

## Audio preprocessing

### Running the script

**Quickstart (default paths – audio in `./audio/YYYY/MM/`):**

```bash
node scripts/build-audio-index.mjs
```

**Full explicit invocation:**

```bash
node scripts/build-audio-index.mjs \
  --input        ./audio \
  --audio-out    ./audio \
  --waveform-out ./waveforms \
  --index-out    ./index.json
```

**Recursive scan (sub-folders included):**

```bash
node scripts/build-audio-index.mjs --recursive
```

**Copy audio files from a raw source folder into the site folder:**

```bash
node scripts/build-audio-index.mjs \
  --input     ./raw-audio \
  --audio-out ./static/audio \
  --copy
```

`--copy` is off by default. Without it the script reads files in place and only
writes `index.json` and waveform JSON files.

**Custom binary paths** (if `ffprobe`/`ffmpeg`/`audiowaveform` are not on `PATH`):

```bash
node scripts/build-audio-index.mjs \
  --ffprobe-bin      ./.bin/ffprobe \
  --ffmpeg-bin       ./.bin/ffmpeg \
  --audiowaveform-bin /opt/homebrew/bin/audiowaveform
```

**Help:**

```bash
node scripts/build-audio-index.mjs --help
```

The script is **idempotent**: re-running it overwrites existing outputs in place
without accumulating stale entries.

### Output schema

`index.json` is a JSON array sorted chronologically (oldest first).

**Audio entry (duration ≥ 5 s):**

```json
{
  "id":            "2026-02-07_17-25-00_barks",
  "filename":      "2026-02-07 17-25-00 barks.m4a",
  "audioPath":     "audio/2026/02/2026-02-07 17-25-00 barks.m4a",
  "waveformPath":  "waveforms/2026/02/2026-02-07_17-25-00_barks.json",
  "date":          "2026-02-07",
  "time":          "17:25",
  "datetimeLocal": "2026-02-07T17:25:00",
  "label":         "barks",
  "durationSec":   18.42,
  "kind":          "audio"
}
```

**Note entry (duration < 5 s or unreadable):**

```json
{
  "id":            "2025-12-11_05-32-00",
  "filename":      "2025-12-11 05-32-00.aac",
  "audioPath":     "audio/2025/12/2025-12-11 05-32-00.aac",
  "waveformPath":  null,
  "date":          "2025-12-11",
  "time":          "05:32",
  "datetimeLocal": "2025-12-11T05:32:00",
  "label":         "",
  "durationSec":   0,
  "kind":          "note"
}
```

| Field | Type | Description |
|---|---|---|
| `id` | string | Stable URL/filename-safe slug (includes seconds) |
| `filename` | string | Original filename |
| `audioPath` | string | Bucket-relative path to the `.m4a` / `.aac` |
| `waveformPath` | string \| null | Bucket-relative path to waveform JSON, or `null` |
| `date` | string | `YYYY-MM-DD` |
| `time` | string | `HH:MM` (display; seconds omitted) |
| `datetimeLocal` | string | `YYYY-MM-DDTHH:MM:SS` (local time, no TZ) |
| `label` | string | Free-text from filename, empty if absent |
| `durationSec` | number | Duration in seconds (3 dp), `0` if unreadable |
| `kind` | `"audio"` \| `"note"` \| `"empty"` | Classification by duration |

**Waveform JSON** (audiowaveform v2 format):

```json
{
  "version": 2, "channels": 1, "sample_rate": 44100,
  "samples_per_pixel": 441, "bits": 8, "length": 4178,
  "data": [-8, 7, -12, 11, "..."]
}
```

`data` contains interleaved `[min, max]` pairs per pixel column.

## Tech stack

| Layer | Technology |
|---|---|
| Frontend framework | SvelteKit 2 + Svelte 5 (runes API) |
| Build / bundler | Vite 6 |
| Static adapter | `@sveltejs/adapter-cloudflare` |
| Asset hosting | MinIO S3 (`s3.angiehjort.com/barktown`) |
| Preprocessing | Node.js ES modules (no npm deps) |
| Audio analysis | ffprobe (duration), audiowaveform (peaks) |
| Styling | Scoped `<style>` per component, no CSS framework |
